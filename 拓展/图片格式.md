
# 基础知识

## 色深

色深指像素点的颜色种类(数据位数), 其单位为 bit(位).  
8位色深即, 2^8 = 256 种颜色.  
目前主流为 24位/1.6千万 色深, 称为真彩色(TrueColor).  
但正在向 10bit(R, G, B 各10bit, 即30位) 发展.  


## 像素点

电子设备其屏幕通过LED等技术来显示颜色, 最小的显示单位即像素点. 依赖于此的图片也由一个个像素点拼组成.  
将图片用"画图"打开, 并放大至 800% 后, 可以看到一个个微小的图片像素点(正方形方块). (100%时, 即图片像素与屏幕像素对齐; 800%, 一个图片像素用 8x8=64 个屏幕像素显示.) 
一般其他图片查看软件放大时会将缺少的部分用过度颜色填报, 而非相同颜色. 所以看不到像素效果.  

- 分辨率(图片尺寸):  

    例: 1920x1080, 表示图片的宽, 高(单位:像素).  
    360P, 720P, 指在标准显示屏(16:9)上高的像素数. 其后的2k, 4k就有些混乱了.   


## RGB

电子屏幕的像素点并不是由一个光源发出的, 而是红(Red), 绿(Green), 蓝(Blue)三个LED灯组成. 其有各种排列方式.  

每个灯都可以发出 0 ~ 255 共 256 个亮度级别的光, 当三个都为255, 即显示白色.  

其三个组合的亮度等级, 就是像素的RGB值, 以此可以混合出色深24位的各种颜色.  

青: (0, 127, 255)  
品红: (255, 0, 127)  


## 透明度

一些图标需要利用透明度, 所以在RGB值后又增加了透明度数值(8bit).  
将透明度称为 Alpha(α).  
像素的属性 RGBA, 称为颜色通道(Color Channel), 透明度即 alpha通道.  


## 像素叠加

RGB_new = RGB1 x α1 + RBG2 x (1 - α1)  
- RGB1 为表层像素, RGB2 为底层像素.  


# 色彩模型


色彩模型指颜色的表示方法, 如 RGB.  
除了RGB外, 还有一些其他的色彩模型: CMYK, HSV, Lab...  

## CMYK

如果说RGB模式是颜色(亮度)的叠加, 那么CMYK就是利用色彩的相减. 光线被物体吸收后再反射出来的颜色成为我们看见的物体颜色, 这就是 **减色色彩模式**.  
RGB适合自发光的屏幕显示器, 而印刷需要减色模式的CMYK.  
CMYK: Cyan(青色), Magenta(品红), Yellow(黄色), Black(黑色).  
> 加入黑色是因为在实际应用中, 难以叠加到真正的黑色, 最多只有褐色.  

![CMYK](../../image/其他/CMYK.webp)  

## HSV/HSL

RGB适用于显示器, CMYK适用于印刷, 但都不适合人来调色.  
所以出现了新的颜色表示方法: HSV(HSB).  

- H(Hue, 色相):  
	按光的波长(即红->紫)的顺序排列为环状, 用 0-360 的数值表示, 0(360)度 对应红色.  

- S(Saturation, 饱和度):  
	代表颜色的浓淡, 范围为 0(淡) - 100(浓).  

- V/B(Value/Brightness, 亮度):  
	代表颜色的亮度, 范围为 0(暗) - 100(亮).  

HSV模型:  
![HSV](../../image/其他/HSV.jpeg)  

对于此模型有人表示, 纯彩色的对应的不应该是白色(即底面边缘对应的亮度不应该是白色, 而应该是灰色).  
所以将底面下放至中间(将顶部中心拉伸), 形成双圆锥:  
![HSL](../../image/其他/HSL.jpg)  
> 其数值仍是 0-100, 新的模型称为 HSL.  
> 最外围的亮度为50, 变成了中灰色.  


现在, HSV一般被平面设计师使用, HSL被网页开发程序员使用.  
因为 图像编辑器 与 网页浏览器 分别不支持对方的模型...  


# 各种图片格式

## BMP 

BMP 全称: Bitmap(位图).  
由微软研发的图片格式.
Bitmap(位图)又是图片唯二类型之一...

- 不支持压缩.  
- 可支持半透明.  


## 略缩图

Thumbs.db文件, 其保存了图片文件的略缩图.  
其大小(内存, 分辨率)都非常小, 能够快速加载和浏览.  
但似乎现在不会以文件形式进行储存.  
可通过磁盘清理进行清除.  


## JPEG

JPG文件常用的后缀有 jpg, jpeg.  

- 有损压缩:  
    JPG文件可将BMP压缩至大约 1/20, 但有一定程度的失真, 在非放大的情况下一般察觉不到.  

- 不支持半透明.    



## PNG

- 无损压缩:  
    png为无损压缩格式.  

- 支持半透明.  


## GIF

GIF格式的特点为动画, 其支持**无损压缩**, **半透明**和**动画**.  
GIF的色深只有 8位, 透明度只有 1位.  

- APNG:  

    png 是抄的 gif, 但没有支持动画, 其后有MPG(已无效), apng(可向后兼容png) 支持动画.  


## TIFF

如BMP, JPG, GIF一般古老的图片格式(1986), 被广泛支持.  
其主要用于 **扫描, 打印, 印刷**, 因为它有其他格式所不支持的特性:  
- 图像模式:  
	像素的格式为CMYK, 而非RGB. 由于打印机只认CMYK, 而临时转换容易产生颜色差异, 所以最好直接使用CMYK格式.  

- 图层:  
	TIFF支持多图层.  

- 多页:  
	图片可以分为多个页面.  

## WebP

webp同时支持**有损**和**无损**两种压缩模式, 且压缩比jpg和png更小.  
webp也支持半透明和动画.  

现在各种浏览器, 图片编辑器, 第三方图片查看器都广泛支持webp格式.  
但windows图片查看器不支持webp格式...  
> PS. win11支持.  


## AVIF

AVIF 是一种由 [开放媒体联盟(AOMedia)](https://aomedia.org/) 维护的图像文件格式，采用了 AV1 视频编解码器的先进压缩技术。  
它可以存储静态和动画图像，文件扩展名为 `.avif`。  
与传统的 JPEG 和 PNG 格式相比，AVIF 文件尺寸更小，但图像质量更高，并支持无损或有损压缩。  
AVIF 格式的关键特点是其卓越的压缩效率。它能够在不牺牲图像质量的前提下，大幅减少文件大小(50%)。  

AVIF格式基于AV1视频解码器, 解码器于2018年3月发布, 9月发布AVIF规范的第一版.  
2020年7月, AVIF格式正式标准化.  
主流的浏览器都支持AVIF格式, win11图片查看器需要安装扩展.  


# 矢量图


矢量图(Scalable Vector Graphics, SVG)是对于位图不同的图片储存模式.  
位图是将图片分割为二维的像素点, 然后储存所有像素点的信息.  
而矢量图通过描述元素的方式记录图像的内容.  
如, 一个圆可以通过 圆心位置+半径 来描述; 当然, 可以再加其他信息, 如颜色, 透明度等.  
类似的, 其他都可以仿照此方式来进行记录.  

SVG使用 xml 格式的源代码来储存文件信息, 文件后缀统一使用svg.  

SVG的优缺点都很明显:  
- 优点:  
	体积小, 无损放大/缩小, 元素可独立编辑(类似位图的自动分层).  

- 缺点:  
	无法描述复杂的图像内容, 如实景照片等.  

SVG一般用于 图标(商标)设计, 工程图绘制...
